ETL Transformation Process

Telecommunications Subscriber Data Integration

ETL Process Flow

EXTRACT

Pull data from Payment System & Sign-up System

→

TRANSFORM

Standardize IDs, Map Account Types, Validate Data

→

LOAD

Insert unified data into Data Warehouse

Source Systems Data Structure

Payment System

Subscriber_Id
STRING (36) - UUID format
Example: 12345678-1234-1234-1234-123456789abc
Subscriber Account Type
STRING (7) - Coded format
PRP-INV, PRP-BUS, PSP-INV, PSP-BUS

Sign-up System

Subscriber_Id
STRING (32) - Hex format
Example: 1234567812341234123456789abc
Payment Model
STRING (15)
Pre-Paid, Post-Paid
Account Type
STRING (30)
Individual, Business
Transformation Process
1
ID Standardization
Remove hyphens from Payment System UUIDs to match Sign-up System format (32 chars)
2
Account Type Mapping
Decode Payment System codes: PRP→Pre-Paid, PSP→Post-Paid, INV→Individual, BUS→Business
3
Data Validation
Validate business rules, check for duplicates, ensure referential integrity
4
Master Record Creation
Merge data from both systems into unified subscriber master record
Data Quality Validation

Duplicate Detection

Identify same subscribers across different ID formats

Referential Integrity

Ensure subscriber records exist in both systems

Business Rules

Validate account type combinations make business sense

Completeness

Check all required fields are populated

Unified Data Warehouse Schema
Master_Subscriber_Id
STRING (32) - Standardized format
Payment_Model
STRING (15) - Pre-Paid/Post-Paid
Account_Type
STRING (30) - Individual/Business
Source_System
STRING (20) - Data lineage tracking
Last_Updated
TIMESTAMP - Change tracking
Data_Quality_Score
INTEGER - Quality metrics
This ETL process ensures a single source of truth for subscriber data while maintaining data quality and business rule compliance.
